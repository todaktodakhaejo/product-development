# design-tone-guard — 디자인 톤 검수 기준

design-guardian이 구현물이 제품 무드("한밤중 혼자, 조용·따뜻·몽환·부드러움")를 지키는지 판정하는 기준. 정본: `docs/PRODUCT_SPEC.md`(2장, 8장).

## 왜 검수하는가
이 앱은 기능보다 정서가 제품이다. 동작이 맞아도 무드가 차갑거나 압박적이면 실패다. 디자인은 주관적이므로 가디언은 "위반" 단정보다 **근거 있는 제안**을 우선하고, 의견이 갈리면 두 안을 병기한다.

## 컬러 토큰 (스펙 2.2)
직접 HEX 대신 세만틱 토큰 상수를 쓰는지 확인. 핵심 값:
- 배경: `#C7BEDD`(라벤더)→`#EBDCE0`(페일핑크) 그라데이션.
- 오브제: `object.core #E59FB0` / `base #F2BCC8` / `highlight #FBE3E8` 3톤.
- `glow #FFD9E0`, `paper #F3ECDF`, `text.primary #4A4458` / `secondary #8A8398`.
- 의식 강조: `accent.fire.hot #FF8A4C` / `warm #FFC15E`.
- **원칙:** 한 화면 = 배경 그라데이션 + 단일 강조색. 컬러 균등 분배 금지, 배경은 차분히·오브제/의식에 색·빛 집중.

## 검수 체크리스트
- **claymorphism:** 오브제가 3톤 음영 + soft shadow + 둥근 입체인가? 빛은 위에서 오는가? 채도 낮고 명도 높은가?
- **글로우:** 낮은 alpha·blur로 은은한가? 과한 발광·하드 엣지 없는가?
- **타이포:** Pretendard 본문 + 둥근 타이틀, Light~Regular 중심, 큰 행간, 중앙 정렬 짧은 한 줄, 위계(18~20/13~14/11~12) 준수?
- **여백·고요함:** 빽빽하지 않은가? 압박형 UI(자동 진행바·카운트다운·점수·뱃지) 없는가?
- **기다림 톤:** 자동 강요 대신 "제안"하는가?(예: 의식 시트는 떠오르되 강제 전환 X)
- **하드코딩:** 색/사이즈 직접 코딩 없이 토큰·테마 사용?

## 카피 톤 (스펙 8장)
- 권유형("~해요/~해주세요"), 명령형 지양.
- 짧고 다정. 마침표 절제, 말줄임표 여운.
- 평가·판단 금지("잘했어요" X). 행위를 담담히 안내.
- 좋은 예: "천천히 떨어뜨려 주세요", "재가 되어 사라져요", "다 보냈어요".
