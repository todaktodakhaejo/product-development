# 감정 오브제 명상 앱 — 기획 · 디자인 명세서 (정본)

> **목적**: Claude Code(Flutter) 핸드오프용. 이미지 없이 이 문서만으로 톤앤매너와 기능을 재현할 수 있도록 작성.
> **플랫폼**: Flutter (iOS / Android). 햅틱·센서 의존도가 높으므로 실기기 테스트 전제.
> **현황 주의**: 현재 `lib/`의 코드는 초기 프로토타입(다크 인디고 "흘려보내기")이며, 이 문서가 목표 제품의 정본이다. 둘은 컨셉·무드·아키텍처가 다르므로, 신규 작업은 이 문서를 기준으로 한다.

---

## 0. 한 줄 컨셉

> **살아있는 디지털 감정 오브제를 만지며 마음을 진정시키고(Soothe) → 떠오르는 생각을 종이에 쏟아낸 뒤(Pour) → 나만의 파괴 의식으로 그 감정을 해소한다(Release).**

ASMR · 말랑이 · 왁뿌볼 · 슬라임 · 반려돌의 촉각적 위안을 디지털 오브제로 옮기고, "복잡할 때 떠오르는 대로 적으면 정리된다"는 인사이트를 메모로, "쏟아낸 감정을 없애고 싶다"는 욕구를 파괴 의식으로 구현한다.

## 1. 제품 개요

### 1.1 핵심 인사이트
- 촉각적 fidget(만지작거림)은 불안을 가라앉힌다 → **오브제 인터랙션**
- 머리가 복잡할 때 떠오르는 대로 쓰면 마음이 안정되고 생각이 정리된다 → **자유 메모**
- 쏟아낸 감정을 물리적으로 없애는 행위에서 카타르시스를 느낀다 → **파괴/해소 의식**

### 1.2 핵심 경험 루프
```
[메인/진정]  ──열기 제스처──▶  [분출/글쓰기]  ──3초 정지──▶  [해소 의식 선택]  ──의식 수행──▶  [마무리]  ──▶  [메인/진정]
   Soothe                          Pour                          Release ritual                  Closing
```
하나의 "의식(ritual)"은 위 한 바퀴를 의미한다. 짧게는 30초, 길게는 수 분.

### 1.3 무드
한밤중 혼자만의 시간. 조용하고, 따뜻하고, 몽환적이며, 부드럽다. 강요하지 않고 천천히 기다려준다. 차갑거나 임상적인 "멘탈케어 앱" 느낌은 피한다.

## 2. 톤앤매너 (디자인 시스템)

### 2.1 무드 키워드
`soft` · `dreamy` · `tactile(촉각적)` · `breathing(숨쉬는)` · `nocturnal(밤)` · `weightless` · `pastel`

전반적으로 **soft 3D / claymorphism** 톤. 오브제는 점토·젤리처럼 부드러운 입체감을 가지며, 배경은 옅은 라벤더–핑크 그라데이션에 은은한 빛 번짐(bloom)이 깔린다. 채도는 낮추고 명도는 높게.

### 2.2 컬러 토큰
세만틱 토큰으로 정의. 다크/딤 분위기지만 배경 자체는 밝은 파스텔이다.

| 토큰 | HEX | 용도 |
|---|---|---|
| `bg.gradient.top` | `#C7BEDD` | 배경 그라데이션 상단 (라벤더) |
| `bg.gradient.bottom` | `#EBDCE0` | 배경 그라데이션 하단 (페일 핑크) |
| `object.core` | `#E59FB0` | 오브제 음영/코어 |
| `object.base` | `#F2BCC8` | 오브제 메인 면 |
| `object.highlight` | `#FBE3E8` | 오브제 하이라이트 |
| `glow` | `#FFD9E0` | 발광·후광·빛 번짐 |
| `paper` | `#F3ECDF` | 메모 종이 (크림) |
| `paper.line` | `#DED3C2` | 종이 결/그림자 |
| `text.primary` | `#4A4458` | 본문 (딥 그레이-퍼플) |
| `text.secondary` | `#8A8398` | 보조 설명 |
| `card.top` | `#9C8FB8` | 해소 의식 카드 배경 상단 |
| `card.bottom` | `#B7ABC9` | 해소 의식 카드 배경 하단 |
| `button.primary` | `#9685B5` | 주 버튼 ("메인으로 돌아가기") |
| `accent.fire.hot` | `#FF8A4C` | 모닥불 코어 |
| `accent.fire.warm` | `#FFC15E` | 모닥불 외곽 |

> **원칙**: 한 화면에 라벤더↔핑크 그라데이션 + 단일 강조색(불꽃 등 의식별)만. 컬러를 균등 분배하지 말고 배경은 차분하게, 오브제/의식에만 색·빛을 집중.

### 2.3 타이포그래피
- **본문/기본**: `Pretendard` (한글 가독성·차분함). google_fonts 미지원이므로 폰트 파일을 `assets/fonts`에 번들.
- **타이틀/감성 카피**: 살짝 둥근 휴머니스트 산세리프 또는 `Gowun Dodum`/`Nanum` 계열로 부드럽게. 너무 굵게 쓰지 않음 (Light~Regular 중심).
- **위계**: 단계 타이틀(예: "진정 단계") 18–20pt / 설명 카피 13–14pt / 상태 미세 카피 11–12pt.
- **정렬**: 감성 카피는 중앙 정렬, 짧은 한 줄 위주. 마침표 최소화.

### 2.4 모션 원칙
- 모든 전환은 **느리고 부드럽게**. 기본 `Curves.easeInOutCubic`, 탄성은 `Curves.elasticOut` 또는 스프링 시뮬레이션.
- 오브제는 항상 미세하게 움직인다(idle breathing). 완전히 멈춘 화면을 만들지 않는다.
- "기다려주는" 인터랙션: 사용자가 멈추면 앱이 다음 단계를 **제안**(자동 강요 X).
- 페이드/블룸을 적극 사용. 하드 컷 전환 금지.

### 2.5 사운드 원칙
- ASMR 지향: 화이트노이즈성 질감음(사락, 뽀드득, 화르륵, 톡).
- 작고 가까운 소리. 무음 모드 대비 **사운드 없이도 햅틱만으로 성립**해야 함.
- 각 제스처/의식마다 1개의 시그니처 사운드.

### 2.6 햅틱 철학 (가장 중요한 차별점)
- 햅틱은 "알림"이 아니라 **촉감의 재현**이다. 시각·청각보다 우선 설계.
- **연속/가변 강도**가 핵심 (예: 흔들기 강도 → 진동 세기). 단발 임팩트로만 끝내지 않는다.
- 사운드 OFF여도 햅틱만으로 오브제가 "살아있다"고 느껴지게.
- 구현 한계(아래 5장 참조): iOS는 Core Haptics로 연속 강도 가능, Android는 amplitude 지원. 기기별 graceful degradation 필수.

## 3. 화면 흐름 (IA)

```
SoothePage (메인·진정)
   │  열기 제스처(양쪽으로 벌리기 / 길게 누르기)
   ▼
PourPage (분출·글쓰기)
   │  3초간 입력 정지 → 하단에 해소 의식 시트 등장
   ▼
ReleaseSelectPage (의식 선택, 6~9종)
   │  의식 선택
   ▼
ReleaseRitualPage (선택한 의식 수행 — 의식별 전용 인터랙션)
   │  의식 완료
   ▼
ClosingPage (마무리)
   │  "메인으로 돌아가기" / 종료
   ▼
SoothePage
```

상태 관리: 한 세션의 메모 텍스트 / 선택된 감정 / 선택된 의식을 하나의 `RitualSession` 모델로 들고 다님.

## 4. 화면별 상세 명세

### 4.1 진정 단계 — SoothePage (오브제 제스처)
카피: **"살아있는 오브제와 교감하며 감정을 진정시켜요"**

부드러운 핑크 젤리 구체가 화면 중앙에 떠 있다. 배경은 라벤더–핑크 그라데이션 + 은은한 빛 번짐. 오브제는 `CustomPainter`로 그린 blob(베지어 제어점 애니메이션 + radial gradient로 입체감 + soft shadow). 모든 제스처는 물리적 변형 + 사운드 + 햅틱 3종 세트로 반응.

| # | 제스처 | 비주얼 반응 | 사운드 | 햅틱 |
|---|---|---|---|---|
| 1 | **기본 상태 (Idle)** | 숨 쉬듯 천천히 부풀고 줄어듦 (0.5~1Hz) | 무음/잔잔 | 없음 (또는 호흡 주기에 맞춘 아주 약한 펄스 옵션) |
| 2 | **한 번 탭** | 톡 눌렸다가 천천히 복원 (~0.6s, elastic) | 통, 통 | light impact 1회 |
| 3 | **길게 누르기** | 누른 만큼 함몰, 떼면 출렁이며 복원 | hum~~~ | 누르는 동안 약한 연속 진동, 뗄 때 릴리즈 펄스 |
| 4 | **문지르기(드래그)** | 손가락 따라 표면 일그러짐, 색이 살짝 번짐 | 사락사락… | 드래그 속도 비례 약한 지속 진동(뽀드득 질감) |
| 5 | **두 손가락 꼬집기** | 양쪽에서 눌려 길쭉해짐 | 끼익— | 꼬집는 거리 비례 긴장 진동 |
| 6 | **흔들기(가속도)** | 화면 안에서 이리저리 팅김 | 데굴, 데굴 | **흔드는 강도 비례 진동 세기 / 벽에 튀길 때마다 임팩트** |
| 7 | **빠르게 돌리기(회전 드래그)** | 팽이처럼 회전 + 잔상 | 위잉~~~ | 회전 속도 비례 연속 진동 |
| 8 | **양손으로 감싸기(뮤트)** | 화면 전체가 어두워지고 오브제만 발광 | 두근.. 두근… | **심장 박동 햅틱 (두-근, 두-근 2연타 패턴 반복)** |

**굴리기 (기기 기울기 기반) 세부**
- 폰 기울기(`accelerometer`)에 따라 오브제가 화면 안에서 굴러다님.
- 굴리기 **시작 순간** 차별화된 진동.
- 부딪치는 **벽 방향**에 맞춰 진동(좌/우 햅틱 뉘앙스).
- **몸 바깥쪽**으로 굴리면 진동 약하게, **몸쪽**으로 굴리면 파도가 밀려오듯 점점 세게.
- 폰을 **던지는 모션**(급가속 후 정지)이면 오브제가 날아가는 느낌으로 반응.

**폰 꽉 쥐기 (옵션, 기기 지원 시)**
- 진동이 점점 세지다가 임계치를 넘으면 "터지는" 효과 후 리셋.
- 압력 센서 부재 시: 길게 누르기 시간 누적으로 대체 가능.

### 4.2 진정 → 분출 전환
오브제 화면에서 글쓰기로 넘어가는 방법 (둘 중 택1 또는 병행 제공):
1. **공을 양쪽으로 잡아 벌리기** → 오브제가 갈라지며 그 사이로 종이가 튀어나온다.
2. **공을 길게 누르고 있기** → 임계 시간 후 종이 화면이 떠오른다.

전환 애니메이션: 오브제가 종이로 morph되거나, 오브제 내부에서 종이가 솟아오르는 연출 + 부드러운 페이드.

### 4.3 분출 단계 — PourPage (글쓰기)
카피: **"마음속 이야기를 종이에 쏟아내요"**

크림색 종이 위에 떠오르는 대로 자유롭게 입력. 정리·교정 부담 없는 "쏟아내기" UX.

- 입력 시작: **위로 스와이프** 또는 **길게 누르기**로 키보드/입력 진입.
- **글이 길어지면 종이가 자동으로 아래로 늘어남** (페이지네이션·스크롤 강제 없음, 무한 두루마리 느낌).
- 타이핑할 때마다 종이가 **미세하게 떨림** (selection-tick급 햅틱 + 시각적 진동).
- **입력을 멈추고 3초간 정지하면** 하단에서 "해소 의식 선택" 시트가 부드럽게 떠오름. (강요 X, 계속 쓰면 사라짐)
- 글자 폰트는 손글씨 느낌 옵션 고려 가능(MVP는 본문 폰트로 충분).

### 4.4 해소 단계 — Release (의식 선택 + 수행)
카피: **"지금 내 감정에 맞는 방식을 선택해요"**

의식 선택 시트는 카드 그리드(보라-그레이 그라데이션 카드). 각 카드 = 의식 이름 + 대응 감정 + 미니 프리뷰. 의식별로 전용 인터랙션 화면을 가진다.

> **MVP 우선순위**: ①②③④를 필수, ⑤⑥은 2차, ⑦⑧⑨는 확장. (아래 7장 참조)

**① 파쇄기로 갈기 — 분노·억울함**
- 종이를 끌어담는 동안 잡았다 놓을 수 있음. 위→아래로 강하게 밀어넣기.
- 파쇄기에 **들어가는 순간 진동**, 갈리는 동안 거친 연속 진동.
- 갈린 종이 조각이 쌓이다가 **폭죽처럼 터지며** 페이드아웃.

**② 구겨서 던지기 — 답답함**
- 두 손가락을 모아 종이를 구김. **세게 모을수록 더 작은 공**이 됨.
- 휘두르거나 스와이프로 던져 날림. 던지는 순간 릴리즈 햅틱.

**③ 모닥불에 태우기 — 슬픔·후회**
- 종이를 천천히 드래그해 불 위에 놓음. **끝부터 천천히 타들어감 (10~15s)**.
- 종이가 **아래에서 위로** 타므로 진동도 **아래→위** 방향감으로. "화르륵" 사운드.
- 재가 되어 사라짐.

**④ 종이 찢기 — 짜증**
- 두 손가락을 벌려 찢기. 자유롭게 여러 번 가능.
- 찢는 순간마다 "찌익" 사운드 + 임팩트 햅틱.
- 위로 스와이프하면 조각들이 흩어져 사라짐.

**⑤ 글자 뒤섞기 — 혼란**
- 폰을 흔들면 글자들이 떨어짐. **더 흔들수록 글자가 부서지고 작아짐**.
- 모래처럼 흘러내려 사라짐. 흔들기 강도 비례 햅틱.

**⑥ 엉킨 실 풀기 — 미련**
- 실 끝을 잡고 천천히 드래그해 풂. **풀리는 매듭마다 "톡, 톡" 햅틱**.
- 다 풀리면 빈 공간만 남음.

**⑦ 비행기 접어 날리기 (확장)**
- 종이가 종이비행기로 접힘. 날리는 모션(폰 스윙/스와이프)을 하면 날아감.

**⑧ 보석함에 넣기 (확장) — 소중히 간직**
- 종이를 보석함에 넣고 뚜껑을 닫으면 보석함 뒤로 후광이 빛남.

**⑨ 저금/골드바 축적 (확장) — 소소한 성취·설렘**
- 파괴가 아닌 **긍정 감정 보관**용. 종이가 금고로 저금되거나 골드바로 쌓임.
- **심장 박동 햅틱**(설렘·떨림) 연출.

> 설계 노트: ①~⑦은 "없애는" 의식, ⑧⑨는 "간직하는" 의식. 부정 감정뿐 아니라 설렘·성취도 다룰 수 있게 양방향 제공.

### 4.5 마무리 — ClosingPage
카피: **"모든 감정을 보낸 후, 다시 나에게 돌아와요"**

- 의식이 끝나면 **3초간 빈 화면** (여백·정적).
- 오브제가 **천천히 다시 등장**.
- **짧은 한 줄 메시지**: "다 보냈어요" (의식별로 카피 변주 가능: 간직 의식이면 "잘 담아뒀어요").
- 하단 **[메인으로 돌아가기]** 버튼 또는 종료.

## 5. 햅틱 & 센서 명세 (구현 레퍼런스)

### 5.1 햅틱 패턴 정의
| 패턴명 | 느낌 | iOS(Core Haptics) | Android(vibration amplitude) | 폴백 |
|---|---|---|---|---|
| `tapPop` | 통 | transient, sharpness↑ | 단발 40ms / amp 128 | `HapticFeedback.lightImpact` |
| `pressHum` | 누름 지속 | continuous, intensity 가변 | 연속 진동, amp 60~120 | 반복 lightImpact |
| `rubTexture` | 사락/뽀드득 | continuous, 속도 비례 | 짧은 펄스 연속 | selectionClick 반복 |
| `shakeBounce` | 벽 튕김 | transient, 강도 비례 | 단발, amp 강도 비례 | mediumImpact |
| `heartbeat` | 두-근 심장박동 | transient×2 (간격 120ms) 주기 800ms 반복 | 패턴 `[0,60,120,60]` 반복 | medium+light 2연타 |
| `shredGrind` | 갈림 | continuous, sharpness↑ noise감 | 강한 연속, amp 200 | heavyImpact 반복 |
| `tear` | 찢김 | transient sharp 1회 | 단발 amp 180 | mediumImpact |
| `burst` | 폭죽/터짐 | transient 강+여운 | amp 255 단발 후 잔진동 | heavyImpact |
| `knotPop` | 톡 (매듭) | transient soft | 단발 amp 90 | selectionClick |

### 5.2 추천 Flutter 패키지
- **`sensors_plus`** — accelerometer / userAccelerometer / gyroscope (흔들기·기울기·굴리기·던지기 모션).
- **iOS 연속/가변 햅틱** — Core Haptics는 Flutter 기본 미지원. 옵션:
  - `flutter_apple_haptics` 또는 직접 **platform channel + CHHapticEngine** (AHAP 패턴) 구현 → 가변 강도/연속 햅틱의 핵심.
- **Android 가변 햅틱** — `vibration` 패키지 (duration / pattern / amplitude 지원, API 26+ amplitude control).
- **기본 임팩트** — Flutter 내장 `HapticFeedback` (light/medium/heavy/selectionClick) → 폴백 및 단발용.
- **물리/모션** — MVP는 Flutter 내장 `SpringSimulation` / `AnimationController` + `CustomPainter`로 충분. 더 풍부한 오브제 물리·파티클(파쇄 조각, 불꽃, 글자 모래)이 필요하면 **`flame`** 엔진 고려.

> **중요**: 기기별 햅틱 역량을 런타임 감지해 graceful degradation. (예: Core Haptics 미지원 → HapticFeedback 폴백, Android amplitude 미지원 → 고정 패턴)

### 5.3 센서 → 인터랙션 매핑
- **흔들기 강도** = `userAccelerometerEvent`의 벡터 크기 → 햅틱 amplitude / 튕김 속도에 정규화 매핑.
- **기울기(굴리기)** = `accelerometerEvent` x·y → 오브제 가속도 벡터.
- **던지기** = 가속도 급상승 후 급하강(피크 임계 초과) 감지.
- **회전** = 화면 제스처(회전 드래그) 우선, 보조로 `gyroscope`.

## 6. Flutter 구현 노트 (아키텍처 제안)

- **렌더링**: 오브제·종이·불꽃·파티클은 `CustomPaint` + `CustomPainter` 중심. 오브제 blob은 제어점 N개의 닫힌 베지어 경로를 `AnimationController`로 흔들고, `RadialGradient` + `BoxShadow`/blur로 claymorphism 입체감.
- **배경**: `Container` `LinearGradient` + 별도 레이어의 blurred radial glow(`ImageFiltered` / `BackdropFilter`).
- **상태관리**: MVP는 `Provider`/`Riverpod` 중 택1. `RitualSession`(memoText, emotion?, ritualType, phase) 단일 모델.
- **라우팅**: 단계별 페이지를 커스텀 `PageRouteBuilder`로 페이드/모프 전환.
- **에셋**: 폰트(Pretendard 등), 사운드(`audioplayers` 또는 `just_audio`)를 `assets`에 번들. 사운드는 짧은 루프/원샷.
- **접근성/설정**: 햅틱 ON/OFF, 사운드 ON/OFF, 모션 민감도 설정 제공. 무음·진동만 모드에서도 전체 경험 성립.

## 7. MVP 범위 정리

### 포함 (MVP)
- 진정: 오브제 idle + 탭 / 길게 누르기 / 문지르기 / 흔들기 / 꼬집기 (최대 5종 제스처) + 햅틱
- 전환: "공 벌려 열기" 또는 "길게 눌러 종이 띄우기" 중 1종
- 분출: 자유 메모(자동 늘어나는 종이) + 3초 정지 시 의식 시트
- 해소: 의식 ①파쇄 ②구겨던지기 ③모닥불 ④찢기 (4종)
- 마무리: 빈 화면 → 오브제 복귀 → 한 줄 메시지 → 메인 복귀

### 2차 / 확장
- 굴리기(기울기 물리), 회전 돌리기, 양손 감싸기(뮤트), 폰 꽉 쥐기
- 의식 ⑤글자 뒤섞기 ⑥실 풀기
- 의식 ⑦비행기 ⑧보석함 ⑨저금/골드바 + 심장박동 햅틱
- iOS Core Haptics 가변 강도 풀구현, 사운드 디자인 고도화
