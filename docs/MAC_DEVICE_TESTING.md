# 맥에서 아이폰 실기기 테스트 — 처음부터 끝까지 가이드

> 이 문서는 **컴퓨터를 잘 몰라도** 따라 할 수 있게 만들었어요.
> 코딩은 윈도우에서 Claude와 함께 하고, **맥은 "아이폰에 앱을 올려서 직접 만져보는 용도"** 로만 씁니다.
> 맥에는 Claude 로그인 안 해요. git으로 코드를 받아서 빌드(앱으로 만들기)만 합니다.

---

## 0. 먼저 알아둘 단어 3개

- **터미널(Terminal)**: 맥에서 글자로 명령을 입력하는 검은 창. 우리가 명령어를 복사해서 붙여넣는 곳이에요.
- **git clone(깃 클론)**: 인터넷(GitHub)에 올라가 있는 우리 프로젝트 코드 전체를 **내 맥으로 통째로 내려받는 것**. 처음 한 번만 해요. 그 다음부터는 `git pull`(새로 바뀐 것만 받기)을 씁니다.
- **flutter run(플러터 런)**: 받아온 코드를 **앱으로 만들어서 연결된 아이폰에 설치하고 실행**하는 명령. 이게 실기기 테스트의 핵심이에요.

---

## 0-1. 터미널 여는 법 (제일 먼저)

1. 키보드에서 `Command(⌘) + Space` 를 누르면 화면 가운데 검색창(스팟라이트)이 떠요.
2. `터미널` 또는 `terminal` 이라고 입력하고 Enter.
3. 검은(또는 흰) 창이 뜨면 성공. **앞으로 나오는 명령어는 이 창에 붙여넣고 Enter** 하면 돼요.

> 💡 붙여넣기는 `Command(⌘) + V`. 명령어 한 줄 붙여넣고 Enter → 끝날 때까지 기다리기 → 다음 줄. 한 번에 한 줄씩!
> 💡 비밀번호를 물어보면 맥 로그인 비밀번호를 입력하세요. **입력해도 화면에 글자가 안 보이는 게 정상**이에요. 그냥 치고 Enter.

## 1. Xcode 설치 (제일 오래 걸림 — 먼저 시작)

**Xcode가 뭐냐면**: 애플이 만든 프로그램으로, 아이폰 앱을 만들려면 반드시 필요해요. 용량이 커서(10GB 이상) **다운로드에 시간이 오래 걸리니 제일 먼저 시작**하세요.

1. 맥에서 **App Store(앱스토어)** 열기.
2. 검색창에 `Xcode` 입력 → 설치(받기) 누르기.
3. 다 받아지면 **Xcode를 한 번 실행**하세요. 처음 실행하면 "추가 구성요소 설치" 같은 창이 뜨는데 **동의/설치** 눌러서 끝까지 진행.
4. (라이선스 동의가 필요할 수 있어요. 터미널에 아래를 붙여넣고 Enter → 비밀번호 입력 → 약관 끝까지 내린 뒤 `agree` 입력):
   ```bash
   sudo xcodebuild -license accept
   ```

> ⏳ 이게 받아지는 동안 아래 2번, 3번을 같이 진행해도 돼요.

---

## 2. Homebrew 설치 (편리한 설치 도우미)

**Homebrew가 뭐냐면**: 맥에서 프로그램을 명령어 한 줄로 깔게 해주는 도우미예요. 이걸 깔아두면 Flutter랑 CocoaPods 설치가 쉬워져요.

터미널에 아래 한 줄을 통째로 복사해 붙여넣고 Enter:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

- 중간에 비밀번호를 물어보면 맥 비밀번호 입력(안 보이는 게 정상).
- 설치가 끝나면 화면에 **"Next steps"** 안내가 나오는데, 거기에 `echo ... >> ...` 로 시작하는 **두 줄**을 따라 입력하라고 할 수 있어요. 그 두 줄을 그대로 복사해 실행하면 됩니다. (Apple Silicon 맥 기준 보통 아래와 같아요 — 화면 안내가 우선):
  ```bash
  echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
  eval "$(/opt/homebrew/bin/brew shellenv)"
  ```
- 확인: `brew --version` 입력 → 버전 숫자가 나오면 성공.

## 3. Flutter 설치

**Flutter가 뭐냐면**: 우리 앱을 만드는 도구예요. 윈도우의 `C:\src\flutter`와는 별개로 맥에도 새로 깔아야 해요. (우리 프로젝트는 Flutter **3.6 이상** 필요)

Homebrew로 설치 (터미널에 붙여넣고 Enter):

```bash
brew install --cask flutter
```

설치 후 확인:

```bash
flutter --version
```

버전 숫자(예: `Flutter 3.27.x`)가 나오면 성공이에요.

---

## 4. CocoaPods 설치

**CocoaPods가 뭐냐면**: 아이폰 앱이 쓰는 부품(라이브러리)들을 챙겨주는 도구예요. iOS 빌드에 필요합니다.

```bash
brew install cocoapods
```

확인:

```bash
pod --version
```

숫자가 나오면 성공.

---

## 4-1. 설치 점검 (flutter doctor)

지금까지 잘 됐는지 한 번에 점검해요:

```bash
flutter doctor
```

- 항목 옆에 **✓(체크)** 가 뜨면 그 부분은 OK.
- ✗ 나 ! 가 뜨면, 그 줄에 **"이렇게 하세요"** 안내가 같이 나와요. 그 안내를 따라 하면 됩니다.
- `Xcode`, `CocoaPods` 항목에 ✓ 가 뜨는지 특히 확인하세요. (안드로이드 항목은 ✗ 여도 우리는 아이폰만 쓰니 괜찮아요.)

> 막히면 이 `flutter doctor` 결과를 **그대로 복사해서 윈도우의 Claude에게 보여주세요.** 어디서 막혔는지 같이 풀어드릴게요.

## 5. 우리 프로젝트 코드 받아오기 (git clone)

이제 GitHub에 있는 우리 코드를 맥으로 내려받습니다. (git은 Xcode 설치 시 같이 깔려요.)

1. 코드를 둘 폴더로 이동 (문서 폴더 안에 받을게요):
   ```bash
   cd ~/Documents
   ```
   > `cd` 는 "이 폴더로 들어가라"는 뜻. `~/Documents` 는 맥의 "문서" 폴더예요.

2. 코드 통째로 내려받기 (이게 **git clone**):
   ```bash
   git clone https://github.com/todaktodakhaejo/product-development.git
   ```
   > GitHub 로그인을 물어볼 수 있어요. 그러면 GitHub 계정으로 로그인(또는 브라우저로 인증)하면 됩니다.

3. 받은 폴더로 들어가기:
   ```bash
   cd product-development
   ```

4. **우리 작업 브랜치로 바꾸기** (중요! main 아니라 우리 파트 브랜치):
   ```bash
   git checkout feat/p1-home
   ```
   > "Switched to branch 'feat/p1-home'" 같은 문구가 나오면 성공.

5. 앱이 쓰는 부품 챙기기:
   ```bash
   flutter pub get
   ```

> 📌 **이건 처음 한 번만** 해요. 다음부터 윈도우에서 새 코드를 올리면, 맥에선 `cd ~/Documents/product-development` 들어가서 `git pull` 한 줄이면 최신 코드가 따라옵니다.

## 6. 아이폰 연결하고 실행하기 (flutter run)

### 6-1. 아이폰 준비
1. 아이폰을 **케이블로 맥에 연결**.
2. 아이폰 화면에 **"이 컴퓨터를 신뢰하시겠습니까?"** → **신뢰** 누르고 아이폰 암호 입력.
3. **개발자 모드 켜기** (iOS 16 이상):
   아이폰 → `설정` → `개인정보 보호 및 보안` → 맨 아래 `개발자 모드` → **켜기** → 아이폰 재시동 → 확인.

### 6-2. 서명(Signing) 한 번 설정 — Xcode에서
앱을 아이폰에 올리려면 "누가 만든 앱인지" 서명이 필요해요. 무료 애플 계정으로 됩니다.

1. 터미널에서 (프로젝트 폴더 안인 상태로):
   ```bash
   open ios/Runner.xcworkspace
   ```
   → Xcode가 열려요.
2. 왼쪽 위에서 파란 **`Runner`** 아이콘 클릭 → 가운데에서 **`Signing & Capabilities`** 탭 클릭.
3. **`Automatically manage signing`** 체크 켜기.
4. **`Team`** 드롭다운 → `Add an Account...` → 본인 **Apple ID 로그인** → 로그인하면 Team 목록에 "(Personal Team)" 이 생겨요. 그걸 선택.
5. (혹시 빨간 오류가 나면 `Bundle Identifier` 를 살짝 바꿔보세요. 예: 뒤에 본인 이니셜 붙이기 → `com.example.emotionResolutionApp.yn`)
6. Xcode는 닫아도 돼요.

### 6-3. 드디어 실행!
터미널에서 (프로젝트 폴더 안):

```bash
flutter run
```

- 연결된 기기를 자동으로 찾아 아이폰에 앱을 설치하고 실행해요. (처음엔 몇 분 걸려요.)
- 아이폰에서 **"신뢰되지 않은 개발자"** 경고가 뜨면:
  아이폰 `설정` → `일반` → `VPN 및 기기 관리` → 본인 Apple ID → **신뢰**.

### 6-4. 코드 고친 걸 바로 반영 (핫 리로드)
`flutter run` 이 돌아가는 터미널 창에서:
- **`r` 키** = 핫 리로드 (바뀐 화면 즉시 반영) ← 제일 많이 씀
- **`R` 키**(대문자) = 완전 다시 시작
- **`q` 키** = 종료

## 7. 평소 반복 흐름 (외워두기)

코딩은 윈도우, 테스트는 맥. 한 바퀴는 이렇게:

1. **(윈도우/Claude)** 코드 수정 → commit → push
2. **(맥)** 터미널에서:
   ```bash
   cd ~/Documents/product-development
   git pull
   ```
3. `flutter run` 이 이미 돌고 있으면 → 터미널에서 **`r`** 눌러 즉시 반영
   (안 돌고 있으면 `flutter run` 다시 입력)
4. **아이폰에서 직접 체험** (흔들기·굴리기·햅틱 등) → 느낀 점을 윈도우의 Claude에게 알려주기
5. 다시 1번으로
